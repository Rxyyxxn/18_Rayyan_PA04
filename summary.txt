[Unit testing]
Unit testing is to test the smallest units or software components
in a program.

Unit testing is the execution of a complete class, routine or small program that has been
written by a single programmer or team of programmers, which is tested in isolation
from the more complete system.

It is type of whitebox testing.

[Intergration Testing]
Integration testing is the combined execution of two or more classes, packages,
components, or subsystems that have been created by multiple programmers or
programming teams.

It may fall under both
white box testing and black box testing.

[System testing]
System testing is the execution of the software in its final configuration, including
integration with other software and hardware systems.

programmers or programming teams.

It is type of blackbox testing

[Acceptance testing]
The acceptance testing plan is created during the design phase and the actual testing is
done after the system is fully developed and tested.

Ideally acceptance testing should be done by customers and end-users

It is type of blackbox testing
